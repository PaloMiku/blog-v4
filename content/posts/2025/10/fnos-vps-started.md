## 前言

个人的娱乐生活最近一直围绕着瞎琢磨项目和玩玩二游，看看动漫。而看动漫这一点对于我这个低付费用户来说又是一件比较捉急的事情，根据三色图定理我并没有在便宜看动漫上收获特别好的体验。

尽管我们有 Animeko 和 Kazumi 这两大神器，但他们带来的体验是参差不齐的，采集站提供的资源要么夹带私货（娱乐城说是），要么只有繁体字幕、要么不够清晰、要么翻译质量令人捉急、当然更多时候还是看它转圈圈。

个人对 BT 和资源站是有一定程度了解的，但是国内 BT 环境不见得多么友好，并且个人也没有全天候让手头设备开机下载上传的能力，网络带宽也不见得多么理想（在学校），也没有那个财力玩实体 NAS 机。

最近每个月例行翻翻 IDC 和 VPS 资料倒是发现了性价比还不错的大盘鸡，2c2g 500g 硬盘到手后简单手动部署了 Caddy 和 qBit 后个人就对这种管理方式感到厌倦了（还是懒），那么有没有什么像 NAS 一样快速管理你的 VPS 的方法呢？还真有，早年个人 DD VPS 的时候曾经见过给 VPS 安装 FnOS 的脚本，只不过当时个人没有合适的大盘鸡所以一直没试过它。

早就听闻 FnOS 也是低价 NAS 不错的选择，这次刚好就试试罢。

## 介绍

首先，FnOS 是一款国产 NAS 系统，当然它也不是纯国产，本质是基于 Debian 的发行版，并且没有开源，所以如果你有很重的开源洁癖，那么接下来也不用看了😮。

![FnOS](https://static2.fnnas.com/official/web/banner.png)

它在它刚诞生的时候还是很火的，很有望作为黑群晖的替代品，当然现在热度也还不错，比黑群晖部署使用简单还没那么吃硬件，所以你也能在小黄鱼上见到几百块甚至一百多块就在跑 FnOS 的玩意😮，当然，不带硬盘甚至风扇都没有。

离题了，FnOS的要求确实很低，系统盘最低只需要占用 8G，常规的 VPS 都可以安装，所以即便是最便宜 VPS 的 15G、20G 硬盘安装 fnOS 也是绰绰有余。

我这个大盘鸡刚好有 15G SSD 固定系统盘，算是绰绰有余了，网络带宽有 1000mbps（当然是共享的），还是可以折腾玩玩的。

在你接下来继续往下看本文之前，我先用几点总结下我用到目前的体验吧：

- 确实懒人救星，作为社区驱动的 NAS 系统在很多常用影视方面，或者其他方面的应用都可以在 FnOS 的应用中心一键安装，当然其实以这些应用的体量自己部署也不是很麻烦，但是我是单纯懒😄，不过 FnOS 是可以使用飞牛 APP 管理的，这点你自己装是不好做到的。
- 结合这套方案我算是实现动漫自由了，小手点点就能自动订阅下载，我只需要等邮件推送给我离线下载的结果，自选字幕组和各种资源。
- FnOS 作为家用 NAS 系统是还可以的，在无公网环境下你可以登录飞牛账号获取免费的 FN Connect 公网穿透服务，大概 2-4 M 带宽，在外面管理设备下达任务和传传小文件是足够的，高速穿透服务的价格不算便宜但也不是抢钱的价格。
- 但对于公网环境来说，尤其没封 80 和 443 的 VPS 商业 IP 来说，FnOS 在网络管理这方面就不能说多么理想了，虽然 FnOS 附带 Nginx 但是用户不能配置它，没有 SSL 证书申请，也没有绑定域名给某个服务，对于商业 IP 使用来说有点浪费。当然对于没有 80 和 443 的家宽 IP 来说倒也无所谓了。

总之就是如果你不是很懂服务运维或者只是像我这样多少懂点但很懒，并且有一个性价比还不错的大盘鸡，你还是可以试试 FnOS 的，省很多事。

## 安装

前面我们就提到过了，有大佬写了 DD 脚本可以快速 DD 安装 FnOS。

脚本 Github 仓库地址：https://github.com/bin456789/reinstall

作为开源且 Star 已经不少的项目来说，安全性应该有一定保证，如果你真的不放心，你也可以 VNC 救援模式挂载自己搓，当然我是不会写这个。一是我真的不会，二是不是所有商家后台都能自己挂载 ISO，虽然你也可以先 VNC 救援系统然后指令下载 FnOS ISO 再自己装，但是还是那句话，我不熟（对不起让你看到废物了我这就走）。

### 下载脚本

当前系统是 Linux 的话（首先你系统装了 curl,wget ），在终端使用以下指令下载 DD 脚本（你是 Windows VPS？你还是自己翻 项目 README 吧，他们给了指令）：

#### 海外服务器

```bash
curl -O https://raw.githubusercontent.com/bin456789/reinstall/main/reinstall.sh || wget -O ${_##*/} $_
```

#### 国内服务器

此处给出的是项目 README 的国内下载指令，理论上你可以换成任何支持 Github 加速的镜像源自己下，当然那不在本文讨论范围之内。

```bash
curl -O https://cnb.cool/bin456789/reinstall/-/git/raw/main/reinstall.sh || wget -O ${_##*/} $_
```

## 执行脚本

执行就很 Easy 了，只需要执行以下指令（PASSWORD 自行换成你想要的 SSH 密码，懂？）并且确定你要的使用的 FnOS 占用系统盘大小（我是直接 15G，视频等数据放 500G 机械）就会自动下载一部分需求文件，在提示你可以重启（Reboot）的时候重启就会自动开始部署，你可以在 后台 VNC 查看进度（不会有机子 VNC 都没有吧？那还是趁早跑路吧）。

```bash
bash reinstall.sh fnos --password PASSWORD
# --password 参数不是必须的，但是提前设 SSH 密码也不是坏事，反正这里就算不用参数你交互脚本也要设密码。
```

## 配置系统

等待 VNC 看到 FnOS Console 了，你就可以访问你`http://你 VPS 的 IP:5666`进入 WebUI 初始化 FnOS 了，因为我已经装了才写的这篇文章，而且懒得再装一遍，所以这里就没有图了，你可以自行判断或者去网上查，实际上交互安装已经很简单了。

配置完我们能够看到赏心悦目的（可道云说是）Web 桌面，别看截图右边内存占用有点高，我这是用了很长时间装了一堆应用并且还在下载工作给你们截的图片，实际上刚装上开机内存占用还是一个可以接受的范围的，1c1g 的机子都可以凑合。

## 影视，下载，刮削

FnOS 本身实际上是一款还在公测的 NAS 系统，但通过安装部署和综合简单体验来说目前可用性已经不错了，我这已经是 0.9 公测版了。

作为我们 NAS 应该的老本行之一，FnOS 获取资源，刮削搭建影视库的体验怎么样呢？

我们可以在软件中心先下载 FnOS 开发的"飞牛影视"，这是我们选择 FnOS 的其中一个关键原因了，它是一个简单可自动刮削的影视库程序，相比 Emby，Jellyfin 等来说，它是开箱即用的。

官方介绍是挂载 NAS，网盘（挂载网盘不是影视提供支持而是 FnOS 文件管理提供支持）文件并自动分类整理和 99% 准确率的自动剐削，用户管理，手机和电视 APP 使用。

当然我实话说确实是还不错的，软件基本很成熟，简单易用，从 TMDB 和 IMDB 刮削影视和动漫数据，并且也可以自动下载和挂载字幕（在线下载字幕的话，动漫字幕缺失的很多其实，自动下载字幕主要还是对影视友好），我是觉得不错的。

那么接下来谈下载资源，FnOS 自带了下载管理器，应该是使用的 qBit 核心，速度不错，但是不能 API 调用很可惜，很多设置也很残废，所以我们如果要 API 调用下载还是得自己安装 Aria2 或者 qBit Web，在应用中心你也可以安装几款主流的下载器，值得注意的是你要在应用设置里好好设置应用能访问读写的目录，不然多半下载因为写入权限不足报错。

不过只是单纯下载种子，这个大概够用了，可以从 NAS 或者用户自己上传种子和磁链，而且支持添加 FnOS NAS 共享链接，支持从其他 FnOS NAS 设备分享快速下载。

对于订阅追番，应用中心同样提供了 Ani Rss 和 AutoBangumi 可选一键安装，还有更多应用，同样别忘了配置好应用可以读写的目录，FnOS 的权限管理还是比较严的，每个应用都有自己的应用空间和应用子用户。

安装好的应用点击打开会直接打开`http://VPS的IP:应用端口号`的应用页面，如果是 FnConnect 是它的专属子域名 （话说你 PC 端干嘛在不在 Web 桌面嵌套浏览器窗口打开呢？挺破坏沉浸感的），如果是飞牛官方应用是桌面打开内部窗口，第三方应用没法主动绑定域名，而配置域名和 IP 的 SSL 证书也麻烦的一批，这算 FnOS 目前还不成熟的部分，这个可能以后会改但肯定不是优先，毕竟大部分 NAS 都没有公网 IP。
