为什么说是博客“真周年随笔”呢？原因也很简单：

>虽然个人折腾博客是从 22 年还在上高中时候开始的，但是变迁无数，更换域名和系统已经可以用两只手来数，博客真正起色还是在上大学以后，所以还是按一年算吧。

不过虽然是上大学以后博客有了正式的起色，但没有过往那堆博客选择和部署经历，也不会让我有现在的经验。

## 过往：探索，选择，更替

在我小学一二年级的时候，家里买了第一台电脑，我那时除了用它玩小游戏和看动画片以外。当时还顺带研究一些新颖的东西，在当时学到的有简单的安卓 APP 构建，简单的网站构建、Photoshop CS3 使用、Windows 维护和重装等，这为我以后选择了自由探索技术打下了一些基础。在那时用的还是比较简单的网站生成器，因为年纪小也没有留下什么有价值的内容。

在初中的时候主要跟小学相比主要是 Photoshop 和接触音乐以及视频剪辑。

自高中起，个人接触到的网络圈子更大了，当时第一个站点是用 PHP 构建的高木资源站，也留下了一些宝贵的回忆。

在此基础上开始接触现代的博客框架，第一个正式博客是基于 WordPress 部署的，使用一个二次元味很足的主题，还有什么音乐播放器，樱花飘散全玩上了，主打一个花里胡哨。

然后不出所料的当时孱弱的虚拟主机带不动这种玩法，于是更换了 Typecho，也没有继续搞十分花里胡哨的东西。但是要说实际内容产出到此时是基本没有的。

22 年底（应该7-9月吧）购入了第一个域名，当时是在 Namesilo 选择的 top 域名。

真正开始有内容产出应该是开始使用 Hexo 的时候。此时已是 23 年初，个人也是在冬天的被窝里写下的第一篇有点价值的文章，应该是 Hexo 和 Github Action 的部署来着，虽然技术含量并不深度，但确实是个人正式成文的文章。

然后在年初于 Porkbun 购入了一个新的 icu 域名，是应用于高木圈的站点（大概），个人把自己少量服务移动到了这里。

在此时期个人算是较深度研究了 WordPress 和 Typecho，接触到了 Halo 博客系统、对 Hexo 的配置使用部署较为熟悉、Markdown 语法基本熟记基础使用，还有 Alist 以及更多小服务的部署等。

23 年中末期个人开始正式接触 Mix Space 系统，在 Dynadot 白嫖的 link 域名上使用它。

24 年各种瞎探索，还有购入了 cn 域名，此时个人已经基本熟悉各路系统部署使用，并且对于静态和 Git 这块相对更为倾斜。在之前有使用 Hexo，Hugo，Next.js，甚至还有纯 HTML 不过刚进入大学时候还是使用 MixSpace 和 WordPress，在个人使用 MixSpace 最久的那段时期对 MixSpace 文档进行了一次贡献风格焕新。

同样是 24 年底，个人购入了现在使用的 com 域名，并且使用 WordPress Oyiso（兜兜转转又回来了说是），算是基本稳定了站点架构，此时才算是正式开始了稳定记录和产出。

## 转变：疑惑，探索，构建

WordPress 虽然很省劲，并且协助我完成了国内备案。但随着使用时间越长，个人不满也是越来越多，从数据库效率问题到插件互相带来的兼容性问题，以及 WordPress 糟糕的 Markdown 支持，小问题逐渐极少成多，让我越来越难忍受它了。

明明是为了省力才选择它的，却为我带来了更多麻烦事。

24 年底到 25 年初个人主力系统更迭到了 Arch Linux，对于代码撰写和构建有了更多的个人理解，25 年春节购入 Steamdeck，个人正式接触 Galgame。

在 25 年初重新迁移回了 MixSpace 系统，它相对来说更加符合我的想法，但是它又不是那么方便。数据库问题和并不算十分完美的体验给我也算是心头挂刺。

3-4 月，个人看到了纸鹿的 Nuxt 博客项目，大受震撼，并且决定彻底转向静态，没过多久就完成了转变，并且在 5 月写下了长文记录个人上半年游玩的 Galgame，纸鹿这套系统个人用着还算顺手。

5 月到 6 月个人写下了很多 Gal 和 Fediverse 的记录文，毕竟自己参与了新的自部署 Sharkey 建设嘛。

然后博客重新归于寂静，在此期间个人更迭主力系统从 Arch Linux 到 Fedora Linux，还有 Vue 和 Nuxt 的了解和学习，玩了更多的 Galgame，和朋友一起玩了不少独立游戏。

暑假个人忙着做兼职，网络上都不怎么活跃了，8 月对博客完成 Nuxt 4 升级。

又是一年开学季，9 月踏入大二，此时个人已经算是在各种服务运维和网站构建之间稍有小成，也打算对网站加入一些自己的想法了。

基于舒适的 Nuxt 4 和先进的 Claude 我完成了对站点功能的升级，写下了更多 Fediverse 和其他服务部署的文章。

## 未来：希望能不忘初心

我希望我未来不会忘记我想要做的事情，我也希望我做这件事情之前是纯真的为了学习探索而进行。

希望自己在之后能了解和学到更多的前端以及服务和站点的运维，也希望自己能有更多新的灵感。

——鬆于 2025.11.04 上午 7 时在温暖的被窝里留（然而暖和不了多久了）
